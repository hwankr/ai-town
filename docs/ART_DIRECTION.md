# ART_DIRECTION

문서 상태: Draft v0.1  
문서 목적: 공간 중심의 타일맵 픽셀 비주얼 기준 정의

## 1. 아트 방향 한 줄 정의

**Gather.town처럼 공간을 걷는 감각**과 **코지 생활 시뮬레이션의 따뜻함**을 결합한,  
관찰하기 좋은 **타일맵 중심 픽셀 마을**을 만든다.

## 2. 비주얼 목표

### 2.1 무엇을 보여 주고 싶은가

- 마을이 실제로 연결된 공간처럼 느껴질 것
- 캐릭터가 작은 픽셀이어도 어디로 가는지 읽힐 것
- 장소마다 생활 목적이 시각적으로 분명할 것
- 같은 공간도 시간대와 점유 인원에 따라 분위기가 달라 보일 것

### 2.2 무엇을 피해야 하는가

- 지나치게 UI 중심으로 보여서 “앱”처럼 보이는 것
- 타일밀도만 높고 동선이 읽히지 않는 혼잡한 맵
- 캐릭터가 배경에 묻혀 이동이나 감정이 안 보이는 것
- 너무 사실적이라 관찰 피로가 큰 화면
- 패널, 카드, 숫자 위주로 공간 화면을 덮는 구성

## 3. 카메라/시점 기준

### 3.1 시점

- 기본은 **탑다운에 가까운 2D 시점**
- 실내는 북쪽 벽 또는 지붕을 생략/투명화해 내부 활동을 읽기 쉽게 처리
- 캐릭터의 머리와 상체는 배경에 묻히지 않도록 대비를 유지

### 3.2 카메라 원칙

- 마을은 한눈에 전부 보이기보다, **현재 사건을 따라가기 좋은 거리감**을 유지한다.
- 기본 줌은 캐릭터 2~4명과 장소 입구 하나가 동시에 보이는 수준으로 시작한다.
- 줌은 정수 배율 위주로 사용해 픽셀 선명도를 유지한다.
- 카메라 이동은 부드럽되, 지나치게 부유하지 않도록 목표 시간이 짧아야 한다.

## 4. 타일/스프라이트 스펙 기본안

### 4.1 월드 단위

| 항목 | 기준 |
| --- | --- |
| 기본 타일 크기 | 16x16 px |
| 권장 렌더 배율 | 3x 기본, 2x/4x 옵션 |
| 캐릭터 캔버스 | 24x32 px |
| 충돌 기준 | 1타일 또는 0.5x1 타일 단위로 단순화 |
| 맵 구성 | 단일 연속 지구 1개를 기본으로 시작 |

16x16 타일은 밀도와 확장성 면에서 유리하고, 24x32 캐릭터는 표정/헤어/아이콘 읽기성을 확보하기 좋다.

### 4.2 레이어 구조

1. Ground Base
2. Ground Detail
3. Collision / Walkable Meta
4. Furniture Low
5. Character
6. Furniture High / Foreground
7. Roof / Wall Fade
8. Lighting Overlay
9. Diegetic FX

## 5. 공간 디자인 원칙

### 5.1 공간은 지도처럼가 아니라 “생활 장면의 무대”처럼 설계한다

좋은 맵은 넓은 것보다 **생활 목적이 명확한 지점**이 많아야 한다.

- 도서관: 책장, 독서 테이블, 조용한 앉기 포인트
- 카페: 카운터, 창가 자리, 2인 좌석, 대기 동선
- 오락실: 기계 앞 시선 정렬, 구경 포인트, 경쟁 장면 포인트
- 공원: 벤치, 산책 루프, 나무 그늘, 만남 분기점
- 주거 구역: 귀가/출발이 보이는 출입구와 창문

### 5.2 동선 우선 설계

- 입구가 어디인지 한눈에 보여야 한다.
- 걷는 경로가 배경 장식에 묻히면 안 된다.
- 캐릭터가 멈추는 지점은 시각적으로 이유가 보여야 한다.
- 장식보다 **서 있기 좋은 자리**와 **만나기 좋은 자리**를 먼저 설계한다.

## 6. 장소별 아트 톤

| 장소 | 키워드 | 시각 포인트 |
| --- | --- | --- |
| 주거 구역 | 편안함, 반복, 사적 공간 | 따뜻한 창문 불빛, 문 앞 매트, 개인 소품 |
| 도서관 | 집중, 조용함, 규칙성 | 책장 리듬, 직선 테이블, 낮은 채도 |
| 카페 | 부드러움, 사교, 머무름 | 컵, 식물, 창가, 따뜻한 목재색 |
| 오락실 | 자극, 리듬, 소음의 암시 | 기계 불빛, 포스터, 강한 포인트 컬러 |
| 공원 | 회복, 개방, 흐름 | 벤치, 나무 그림자, 곡선 산책로 |
| 확장 부지 | 기대감, 가능성 | 가림막, 공사 표지, 빈 공간의 여지 |

## 7. 캐릭터 아트 기준

### 7.1 실루엣 우선

캐릭터는 작아도 구분 가능해야 한다.

- 헤어 실루엣 차이
- 상의/하의 색 대비
- 액세서리 1개 정도
- 각 캐릭터를 대표하는 보조색 1개

### 7.2 애니메이션 최소 세트

| 종류 | 필요도 | 설명 |
| --- | --- | --- |
| Idle | 필수 | 호흡, 시선, 미세 흔들림 |
| Walk 4방향 | 필수 | 이동 가독성 핵심 |
| Sit | 필수 | 카페/도서관/공원에서 사용 |
| Interact | 필수 | 책 보기, 게임하기, 대화 제스처 |
| Emote Pop | 필수 | 감정/관심/반응 표시 |
| Special Loop | 선택 | 캐릭터 개성용 (예: 스트레칭, 허밍) |

### 7.3 표정 표현

초기에는 얼굴 픽셀이 적으므로 과도한 표정보다 아래 수단을 우선 사용한다.

- 머리 위 아이콘
- 몸 각도/시선 방향
- 짧은 제스처
- 말풍선 꼬리/말줄임
- 주변 소품 반응

## 8. 색채 기준

### 8.1 팔레트 원칙

- 전체 톤은 **따뜻하고 부드럽게**
- 포인트 컬러는 장소 성격을 구분하는 용도로만 제한적으로 사용
- 배경 대비가 너무 세면 캐릭터가 묻히므로, 배경 채도는 한 단계 낮춘다
- UI는 아트보다 튀면 안 된다

### 8.2 시간대별 라이팅

| 시간대 | 연출 |
| --- | --- |
| 아침 | 맑고 중성적인 광량 |
| 낮 | 가장 읽기 쉬운 명도, 활동성 강조 |
| 저녁 | 주황/분홍 계열의 따뜻한 편광 |
| 밤 | 실내 조명 대비, 외부는 차분한 블루 톤 |
| 심야 | 조용한 저채도, 창문 조명 포인트 |

## 9. UI와 아트의 관계

### 9.1 UI 원칙

- UI는 화면을 덮지 않고 가장자리 또는 소형 오버레이로 남는다.
- 정보는 가급적 월드 안에서 먼저 보여 준다.
- 숫자/바보다 **위치, 행동, 아이콘, 말풍선**을 우선한다.

### 9.2 허용되는 UI

- 우하단 또는 측면의 짧은 입력 패널
- 선택 캐릭터 소형 카드
- 배속/일시정지
- 작은 알림 토스트
- 해금 알림
- 일기/요약 팝업(짧게)

### 9.3 지양할 UI

- 전체 화면 관리자 대시보드
- 항상 열려 있는 대형 정보 패널
- 복수 캐릭터의 수치를 동시에 펼쳐 놓는 운영 화면
- 표/그래프 중심 메인 화면

## 10. 맵 구성 기본안

### 10.1 핵심 맵 목표

초기 맵은 “작지만 바쁜 동네”여야 한다.  
넓기보다 다음이 중요하다.

- 장소 간 이동 시간이 지루하지 않을 것
- 캐릭터들이 자주 스쳐 지나갈 것
- 카메라가 과도하게 멀리 이동하지 않아도 장면이 이어질 것
- 확장 부지가 이미 보이거나 가까이 있어 미래감을 줄 것

### 10.2 추천 구성

- 중심 골목
- 카페 앞 교차점
- 도서관 입구
- 공원 진입부
- 오락실 외벽
- 주거 구역 골목
- 확장 부지 경계

이렇게 **“마주침 포인트”**가 반복적으로 생기는 구조를 우선한다.

## 11. VFX/연출 기준

- 과한 파티클보다 작은 생활 FX 우선
- 발걸음, 문 열림, 작은 먼지, 조명 깜빡임 정도면 충분
- 이벤트 발생 시 카메라, 말풍선, 아이콘, 아주 짧은 사운드로 리듬을 준다
- LLM 연동 대사/일기는 화려한 컷신보다 **짧고 잔잔한 연출**이 어울린다

## 14. 품질 바

이 프로젝트에서 좋은 아트는 “고해상도”가 아니라 다음 조건을 만족하는 아트다.

- 캐릭터가 어디로 가는지 잘 보인다.
- 공간이 생활 장소처럼 읽힌다.
- 동일한 장소도 시간대와 점유도에 따라 다르게 보인다.
- UI가 마을보다 더 강하게 보이지 않는다.
- 구경만 해도 작은 이야기가 느껴진다.
