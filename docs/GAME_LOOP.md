# GAME_LOOP

문서 상태: Draft v0.1  
문서 목적: 시뮬레이션 중심 게임의 실제 플레이 루프와 시간 구조 정의

## 1. 게임 루프 요약

이 게임의 루프는 전통적인 “퀘스트 수행 → 보상 획득” 구조보다, 다음 4개 고리로 설명하는 것이 맞다.

1. **관찰 루프**: 캐릭터가 움직이고 만나고 흩어지는 모습을 본다.
2. **입력 루프**: 사용자가 자신의 행동을 짧게 입력한다.
3. **반영 루프**: 입력이 일정, 욕구, 장소 선호, 관계에 반영된다.
4. **성장 루프**: 누적 결과로 캐릭터/시설/장소가 해금된다.

이 루프의 핵심은 조작량이 아니라 **인과를 보는 즐거움**이다.

## 2. 시간 축 설계

### 2.1 기본 시간 규칙

- 기본 배속 1x 기준: **현실 1초 = 게임 1분**
- 게임 1일: **현실 24분**
- 내부 고정 틱: **초당 4틱**
- 1틱의 게임 시간: **15초**
- 지원 배속: 1x / 2x / 4x
- 일시정지 지원
- 백그라운드 오프라인 진행은 초기 버전에서 제한적으로만 지원하고, 우선은 앱이 켜져 있을 때 시뮬레이션이 흐르는 구조를 기본으로 한다.

이 설정은 “구경 가능한 속도”와 “하루가 너무 길지 않음” 사이의 절충안이다.

### 2.2 시간대 구간

| 시간대 | 게임 시간 | 목적 |
| --- | --- | --- |
| 아침 | 06:00~10:00 | 기상, 이동 시작, 하루의 초기 루틴 형성 |
| 낮 | 10:00~16:00 | 공부/일/활동 중심, 사용자 입력 반영이 가장 잘 보임 |
| 저녁 | 16:00~21:00 | 사교, 카페/오락실/공원 장면 밀도 증가 |
| 밤 | 21:00~02:00 | 귀가, 감정 정리, 일기/회고 톤 강화 |
| 심야 | 02:00~06:00 | 수면, 정적 연출, 다음 날 준비 |

## 3. 세션 루프

### 3.1 1회 세션의 기본 흐름

1. 앱 실행 후 바로 마을 화면 진입
2. 현재 가장 흥미로운 장면 또는 선택 캐릭터로 카메라 포커스
3. 사용자는 마을을 지켜보다가 필요할 때만 짧게 생활 입력
4. 입력에 따라 캐릭터 행동과 장면 변화 발생
5. 짧은 관계 이벤트, 장소 이동, 감정 연출이 연쇄적으로 발생
6. 하루 종료 또는 세션 종료 시 저장, 요약, 다음 해금 힌트 제공

### 3.2 기대 세션 길이

| 세션 타입 | 길이 | 기대 경험 |
| --- | --- | --- |
| 짧은 체크인 | 5~10분 | 변화 확인, 입력 1~2회, 장면 1~2개 관찰 |
| 표준 세션 | 15~30분 | 하루 절반~1일 관찰, 입력 2~4회, 이벤트와 진행 확인 |
| 장기 방치형 관찰 | 30분 이상 | 여러 장면 누적 관찰, 관계/장소 사용 패턴 변화 체감 |

## 4. 순간 루프 (Moment-to-Moment)

플레이어가 화면에서 느끼는 기본 리듬은 다음과 같다.

1. 누군가 이동한다.
2. 누군가 멈춘다.
3. 상호작용 가능한 상대나 장소를 만난다.
4. 짧은 감정/행동 결과가 나온다.
5. 다시 새로운 의도로 움직인다.

이 루프가 단조롭지 않으려면 화면에서 **15~30초마다 최소 1회 시선 이동 포인트**가 생겨야 한다.  
시선 이동 포인트는 아래 중 하나다.

- 새로운 장소로 향하는 이동
- 만남/충돌/합류
- 감정 아이콘/짧은 대사
- 이벤트 포커스
- 장소 상태 변화(불 켜짐, 붐빔, 비어 있음)
- 해금 예고 또는 공지

## 5. 캐릭터 자율 루프

각 캐릭터는 완전 랜덤이 아니라, **일정 기반 + 욕구 기반 + 관계 기반 + 상황 기반** 혼합 구조로 움직인다.

### 5.1 자율 루프 단계

| 단계 | 설명 | 주기 |
| --- | --- | --- |
| Routine Plan | 하루의 큰 블록(집, 공부, 휴식, 놀이 등)을 배치 | 하루 시작, 큰 변화 시 재계산 |
| Intent Selection | 지금 가능한 행동 후보를 평가해 현재 의도를 고름 | 1~4틱마다 또는 트리거 발생 시 |
| Navigation | 목적지까지 경로 계산 및 이동 | 필요 시 |
| Action Execution | 책 읽기, 앉기, 대화, 휴식, 구경 등 실행 | 행동 길이에 따라 |
| Social Check | 주변 인물/관계/장소 맥락으로 이벤트 가능성 판정 | 같은 장소 또는 근접 시 |
| Memory Update | 결과를 기억과 관계에 반영 | 행동 종료 시 |

### 5.2 행동 후보 예시

- 도서관으로 이동
- 카페에서 앉기
- 공원 산책
- 오락실 기계 앞에 서기
- 특정 캐릭터에게 접근
- 피곤해서 집으로 귀가
- 긴장도가 높아 혼자 있고 싶어짐
- 사용자가 입력한 활동에 동조해 장소 우선순위 변경

## 6. 사용자 입력 루프

### 6.1 입력 구조

사용자는 다음과 같은 행동을 직접 입력한다.

- 공부
- 게임
- 운동
- 휴식
- 일/작업
- 사교
- 창작
- 기타 메모형 입력(후순위)

초기 버전에서는 **공부 / 게임 / 운동 / 휴식** 4종으로 시작한다.

### 6.2 입력 후 반영 순서

1. 사용자가 활동 카테고리와 대략적 시간을 입력한다.
2. 입력은 `PlayerActionEntry`로 저장된다.
3. 시스템은 이를 `InfluenceSignal`로 변환한다.
4. 관련 캐릭터의 욕구/장소 선호/루틴 우선순위가 수정된다.
5. 필요 시 당일 일정 일부가 재계산된다.
6. 다음 몇 분 안에 가시적 행동 변화가 나타난다.
7. 하루 단위 누적값은 해금/관계/장기 성향 변화를 만든다.

### 6.3 즉시성 효과와 누적 효과

| 효과 종류 | 시간 범위 | 예시 |
| --- | --- | --- |
| 즉시성 효과 | 입력 후 수 분 | A가 카페 대신 도서관으로 향함 |
| 단기 누적 효과 | 당일 | 공부 입력이 쌓여 집중 행동 비율 증가 |
| 중기 누적 효과 | 며칠 | 특정 장소의 점유율이 높아지고 신규 이벤트가 열림 |
| 장기 누적 효과 | 여러 세션 | 캐릭터 해금, 시설 확장, 대사 톤 변화 |

## 7. 관계 루프

이 게임은 단순히 장소를 돌기만 하는 NPC가 아니라, 관계가 누적되며 행동이 달라지는 주민이 필요하다.

### 7.1 관계 축

초기 버전에서는 아래 4축을 기본으로 둔다.

- 친숙도
- 신뢰
- 호감
- 마찰

### 7.2 관계가 가시적으로 드러나는 방법

- 먼저 다가가거나 피하는 이동 패턴
- 같은 장소를 선호하게 되거나 피하게 됨
- 짧은 대사/이모트 차이
- 함께 있는 시간 증가/감소
- 이벤트 우선순위 변화
- 해금 조건 충족

### 7.3 관계 이벤트 발생 조건 예시

- 같은 장소 + 서로의 현재 감정이 양호 + 과거 마찰이 낮음
- 특정 장소에서만 가능한 이벤트 (카페 대화, 도서관 동석, 공원 산책)
- 사용자 입력의 영향으로 같은 시간대에 같은 장소를 쓰게 됨
- 한 캐릭터가 다른 캐릭터를 찾는 의도를 선택함

## 8. 하루 루프

### 8.1 하루 시작

- 전날 누적 상태 반영
- 수면/회복 결과 반영
- 일정 블록 초안 생성
- 해금 조건 일부 평가
- 날씨/특수 연출은 후순위

### 8.2 하루 중반

- 사용자의 입력이 가장 자주 개입하는 구간
- 공부/운동/게임/휴식이 장소 이용 패턴과 사회적 만남에 반영
- 핵심 장면 밀도가 높아야 하는 구간

### 8.3 하루 종료

- 귀가
- 당일 관계 변화 정산
- 기억/감정 잔상 반영
- 짧은 일기/하이라이트 생성
- 스냅샷 저장

## 9. 장기 루프

### 9.1 3~5세션 루프

- char_b 해금
- 오락실/카페 장면 다변화
- 장소 선호 패턴이 눈에 띄게 갈림
- A의 생활 리듬이 플레이어 입력에 따라 안정형/산만형 등으로 달라짐

### 9.2 1~2주 루프

- char_c 해금
- 확장 부지 개방
- 컴퓨터실/노래방 등 신규 시설 중 1개 이상 설치
- 관계 조합별 이벤트 볼륨 증가
- 마을의 시간대별 분위기가 더 뚜렷해짐

### 9.3 장기 루프의 목적

장기 루프는 숫자 성장보다 **마을의 밀도와 다양성**이 커지는 감각을 만들어야 한다.  
사용자가 돌아왔을 때 “새로운 장면이 생겼다”가 가장 중요한 보상이다.

## 10. 지루함 방지 규칙

### 10.1 시각적 공백 방지

- 같은 화면 영역에 45초 이상 아무 일도 없으면 안 된다.
- 선택 캐릭터가 장시간 혼자 있을 경우, 다음 의미 있는 장소 이동을 유도한다.
- 이벤트가 너무 산발적이면 카메라 자동 포커스가 장면을 보조한다.

### 10.2 행동 다양성 방지선

- 같은 행동이 연속 3회 이상 반복되면 점수 감쇠
- 같은 장소 체류가 과도하면 대체 행동 가중치 증가
- 특정 관계 조합이 과도하게 반복되면 이벤트 템플릿 쿨다운 적용

### 10.3 관찰 보상 강화

- 행동의 결과는 숫자보다 장면으로 먼저 보여 준다.
- 가벼운 이모트, 머리 위 아이콘, 짧은 말풍선, 자리 바꿈 등 작은 피드백을 자주 사용한다.
- 큰 이벤트만 기다리게 하지 않고 작은 장면을 계속 발생시킨다.

## 11. 실패 상태와 긴장감

이 게임은 전통적 의미의 “패배”가 중심이 아니다.  
대신 아래와 같은 **소프트 실패/소프트 드리프트**를 둔다.

- 피곤이 누적되어 활동 반경이 줄어듦
- 관계 마찰이 높아져 특정 만남이 줄어듦
- 장소 이용이 단조로워져 해금 속도가 느려짐
- 휴식 부족으로 저녁 이벤트 참여율이 낮아짐

중요한 점은 이런 상태가 **회복 가능**해야 한다는 것이다.  
휴식, 새로운 입력 패턴, 장소 확장, 시간이 지나면서 회복할 수 있어야 한다.

## 12. 카메라와 루프의 관계

시뮬레이션이 좋아도 플레이어가 보지 못하면 재미가 없다.  
따라서 카메라는 단순 시점 장치가 아니라 게임 루프의 일부다.

### 12.1 카메라 모드

- 자유 이동
- 선택 캐릭터 추적
- 이벤트 자동 포커스
- 하이브리드 포커스(기본 추적 + 중요 이벤트 시 살짝 이동)

### 12.2 카메라 설계 원칙

- 자동 이동은 부드럽고 짧아야 한다.
- 사용자가 손으로 카메라를 움직이면 자동 포커스를 잠시 해제한다.
- 세계 전체보다 “지금 재밌는 장면”을 보여 주는 것이 우선이다.

## 13. 첫 20분 플레이 기대 흐름

| 구간 | 기대 장면 |
| --- | --- |
| 0~2분 | A가 하루를 시작하고 첫 장소로 이동 |
| 2~5분 | 사용자가 첫 입력을 넣고 관련 장소 변화가 보임 |
| 5~8분 | 카페 또는 도서관에서 짧은 만남/아이콘/대사 발생 |
| 8~12분 | 이동 패턴 차이, 공원/오락실 쪽 분기 발생 |
| 12~16분 | 관계 변화가 작은 장면으로 드러남 |
| 16~20분 | 해금 힌트 또는 하루 회고 장면 노출 |

## 14. 루프 검증 체크리스트

- 사용자가 아무 입력도 하지 않아도 10분간 볼거리가 유지되는가?
- 입력 1회가 실제 장면 차이를 만들었는가?
- 캐릭터가 “왜 저기로 가는지” 설명 가능한 상태인가?
- 하루의 흐름이 아침/낮/저녁으로 체감되는가?
- 관계 변화가 숫자 UI가 아니라 행동으로 드러나는가?
- 세션 종료 시 다음에 다시 켜 보고 싶은 작은 갈고리가 남는가?
